## 背景

经过性能测试（详见 #3），发现：
- CPU: 1.64ms/文本
- GPU: 1.40ms/文本（仅快 17%）
- NPU: 无法使用（驱动不支持动态形状）

## 决策

**简化系统，移除 GPU/NPU 支持，仅使用 CPU**

## 原因

1. **性能足够**: CPU 处理单条笔记只需 ~1.6ms，完全满足实时需求
2. **降低复杂度**: 
   - 移除 optimum 依赖
   - 移除复杂的设备检测和回退逻辑
   - 移除 GPU/NPU 相关的错误处理
3. **提高稳定性**: 减少依赖和环境问题
4. **简化安装**: 不需要处理 Windows 长路径等问题

## 改动内容

- [ ] 简化 `npu_backend.py` -> `embedding_backend.py`
- [ ] 移除 optimum 依赖
- [ ] 更新 requirements.txt

## 性能基准

| 指标 | 数值 |
|------|------|
| 单条编码 | ~1.6ms |
| 批量 100 条 | ~0.16s |
| 每秒处理 | ~600 条 |

完全满足个人知识管理需求。
